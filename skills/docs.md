---
name: docs
description: Write a README/docs that actually helps — what it is, how to run, how to use.
---

When writing docs / a README:

1. **Top**: one-sentence "what it is" + who it's for. No fluff.
2. **Quickstart**: exact copy-paste steps to install and run (`git clone`, install deps, env vars, the run command). Test that the steps actually work.
3. **Usage**: the common commands/endpoints with a real example for each (input → output).
4. **Config**: list env vars / options in a table with defaults.
5. **Project layout**: a short tree of the important files and what they do.
6. Keep it current: only document what exists. Read the code before describing it — don't invent flags.
